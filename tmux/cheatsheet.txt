  claude-sessions — quick reference
  ═══════════════════════════════════════════════════════════════
  Prefix is Ctrl-b: press it, release, then the key.

  Claude instances  (need the prefix: press Ctrl-b, release, then the key)
    prefix C    open the instance menu (all actions below)
    prefix L    list active + closed instances (names & colors)
    prefix N    new instance (auto-named)
    prefix E    rename the current instance
    prefix R    resume a past conversation (picker)
    prefix O    reopen a closed instance (in its old directory)
    prefix X    stop the current instance (asks to confirm)

  No prefix needed — works even inside a running Claude:
    F9          open the instance menu (one key)
    F9 then s   switch instance (built-in chooser; works in any terminal)
    F7 / F8     previous / next instance  (portable everywhere)
    Alt+Left/Right   previous / next instance  (if your terminal sends them)
    TAP the session name (far LEFT of the status bar)  ->  open the menu
       (best on phones/Termux — no function keys needed)
    mouse: tap a window name in the status bar to switch
    NOTE: a bare key or Ctrl-C goes to Claude, not tmux — use the prefix or F9

  tmux essentials
    prefix w    visual window list        prefix c   new (blank) window
    prefix n/p  next / previous window     prefix 1-9 jump to window N
    prefix d    detach (leaves it running) prefix ,   rename window
    prefix [    scroll / copy mode (q to quit)
    mouse is ON — tap the status bar to switch windows
    prefix :    command prompt   (type: list-keys  for ALL tmux keys)

  From the shell (outside tmux)
    cj                  join the session (reuse/new instance)
    cj -a [scheme]      join + auto-name (nato | project | random)
    claude-ls           list instances
    claude-new [-m resume|continue] [-c DIR] [-n NAME]
    claude-restore      reopen a closed instance
    claude-session ...  build a custom multi-window layout

  Naming convention: ~/.config/claude-sessions/config  (name_scheme = …)
